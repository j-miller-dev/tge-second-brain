# VS Code Extensions Guide

This guide explains the recommended Markdown and productivity extensions, including when and why to use each one.

## 1. Markdown Lint

- Purpose: Enforces consistent Markdown style.
- Use it when: You want clean, readable Markdown files with fewer formatting mistakes.
- Why it helps: It catches issues like inconsistent headings, spacing, list formatting, and heading levels.
- Best for: Writing notes, documentation, and knowledge-base pages.

## 2. Markdown Preview Mermaid Support

- Purpose: Enables Mermaid diagrams inside Markdown preview.
- Use it when: You want to draw flowcharts, sequence diagrams, or simple architecture diagrams in notes.
- Why it helps: You can visualize workflows, study maps, or decision trees directly in Markdown.
- Best for: Process notes, project planning, and concept mapping.

## 3. Markdown Preview Enhanced

- Purpose: Improves the Markdown preview experience in VS Code.
- Use it when: You want a richer preview with better styling and more features.
- Why it helps: It makes Markdown feel closer to a polished document viewer.
- Best for: Reading notes, writing documentation, and previewing formatted content.

## 4. Markdown All in One

- Purpose: Adds useful Markdown shortcuts and utilities in one extension.
- Use it when: You write a lot of Markdown and want faster formatting workflows.
- Why it helps: It improves shortcuts, table support, list handling, and general editing experience.
- Best for: Daily note writing, docs, and knowledge management.

## 5. GitLens Supercharged

- Purpose: Enhances Git visibility inside VS Code.
- Use it when: You want to see who changed what, when, and why.
- Why it helps: It adds blame annotations, commit history, and repository insights directly in your files.
- Best for: Tracking work notes, collaborating, and reviewing changes over time.

## 6. Foam

- Purpose: Adds Obsidian-style linking and knowledge management features to VS Code.
- Use it when: You want wiki-style links, backlinks, and note networks without Obsidian.
- Why it helps: It allows you to build a connected personal knowledge base inside VS Code.
- Best for: Second-brain note systems, research notes, and project thinking.

## 7. Code Spell Checker

- Purpose: Detects spelling mistakes in comments, Markdown, and code.
- Use it when: You want fewer typos in notes and documentation.
- Why it helps: It improves clarity and professionalism in writing.
- Best for: Journals, study notes, and polished documentation.

## Recommended Setup

If you want a simple and useful setup for this vault, start with:

- Markdown Lint
- Markdown Preview Enhanced
- Markdown All in One
- Foam
- GitLens Supercharged
- Code Spell Checker

Add Mermaid support when you want to visualize ideas or processes.
