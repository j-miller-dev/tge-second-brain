# My Note-Taking System — Context for AI Assistance

Use this as context whenever I ask for help drafting, reviewing, or linking notes in my Obsidian vault. Hold me to this, don't just be agreeable.

## The method (Ahrens' Zettelkasten, four layers)

1. **Fleeting** — quick captures, temporary, cleared out regularly. Never help me leave these lying around long-term.
2. **Literature notes** — one per source (book/lecture/article). Written in my own words, tied to that source, can be as long as needed. This is where a resource's key claims live.
3. **Permanent notes** — atomic (one idea per note), written for my future self with zero context needed, densely linked to other permanent notes via `[[wikilinks]]`. This is the actual compounding layer.
4. **Projects** — plans, checklists, trackers. Not knowledge. Never mix these into the Permanent layer.

## What "good" looks like in a Permanent note

- **One idea only.** If a note needs "and" to summarise it, it's two notes.
- **Plain language, my own words.** Not a paraphrase of the source with the serial numbers filed off — a genuine restatement, as if explaining it to someone with no context.
- **Self-contained.** No "as mentioned above" or relying on surrounding notes — this note should make sense cold, months from now.
- **Densely linked.** Actively look for connections to existing notes, especially across domains (e.g. a training-philosophy idea and a dev-practice idea that share a mechanism). A note with zero links is a smell.
- **Short.** A paragraph or two, not a page. If I'm writing a page, I'm probably still doing a literature note, not a permanent one.

## What I want from AI help specifically

- **Push back if a note isn't atomic** — suggest splitting it rather than letting it slide.
- **Suggest links** to concepts I've likely already written about, based on what's in my vault, rather than assuming a note stands alone.
- **Flag when I'm just restating the source** rather than synthesising it — that's still a literature note, not a permanent one, even if it's in my own phrasing.
- **Don't write the note for me.** Ask me to explain the idea back first if I'm stuck — that struggle is the point (see: writing forces understanding, doesn't just record it).
- **Keep style consistent**: informal but precise, short sentences, concrete examples over abstract description where possible.

## Naming and structure

- Permanent notes: title is the core claim or concept, not a source name (e.g. `Closures Capture Variables, Not Values`, not `Notes on Chapter 4`).
- Literature notes: titled after the source (e.g. `Epp — Discrete Mathematics with Applications`).
- Hybrid notes are allowed when a note is partly source-based and partly conceptual: keep the source context in the note body or frontmatter, but still aim for one main idea per note.
- MOC (Map of Content) notes exist per domain (e.g. career/curriculum, health/training) and link out to relevant Permanent + Literature notes — they're the browsing layer, not a storage layer.
- Tags mark domain (`#career`, `#fitness`, `#bjj`), not note type — note type is structural (folder/frontmatter), not a tag.
