# Second Brain in VS Code

This folder now serves as a lightweight, Git-backed knowledge base for work, reading, and study.

## Structure
- Inbox/ - fleeting notes and quick capture
- Literature Notes/ - lecture notes, chapter notes, and source-based summaries
- Permanent Notes/ - atomic ideas and standalone insights
- Projects/ - active plans, checklists, and trackers
- Work/ - work notes and decision logs
- Kindle/ - book highlights and reading summaries
- Discrete Math/ - domain hub for discrete math study notes
- Templates/ - reusable note templates
- Assets/ - images and other attachments

## Suggested workflow
1. Capture ideas in Inbox.
2. Move source-based material into Literature Notes.
3. Extract atomic ideas into Permanent Notes.
4. Keep active plans in Projects.
5. Commit changes regularly so you have a backup and history.

## Note conventions
- Literature notes are tied to a source and written in your own words.
- Permanent notes are short, self-contained, and linked to other notes.
- Hybrid notes are fine when a note needs both source context and a compact conceptual point; keep the core idea clear and avoid turning the note into a dump.
- Projects are not the same as knowledge notes; keep them separate.
